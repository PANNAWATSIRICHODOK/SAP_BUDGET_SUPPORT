Summary

Login เข้าแล้ว Budget ทำงานทันที

เราใส่ INSERT INTO "UAT_JBF"."NDBS_LOGIN_TRACE" ไว้ใน NDBS_BUDGET_CONTROL และ NDBS_UpdateAllBudgetAmount เพื่อดูว่า add-on ถูกเรียกเมื่อใด จากนั้นล้าง table แล้ว login เข้า SAP โดยยังไม่เปิดเอกสารหรือเริ่ม transaction ใด ๆ แต่ใน NDBS_LOGIN_TRACE กลับมีข้อมูลถูก insert เข้ามาทันที จึงยืนยันได้ว่า Budget ถูกเรียกตั้งแต่ช่วง login จริง และใน export ล่าสุด helper ปลายทางถูก rewrite เป็น set-based แล้ว ดังนั้นต้องแยกให้ออกว่า flow ยังถูกเรียกเหมือนเดิม แต่ คอขวดเดิมถูกแก้ไปแล้ว

สิ่งที่พิสูจน์ได้จากการทดสอบนี้

  • แค่ login เข้า SAP ก็มีการเรียก NDBS_BUDGET_CONTROL
  • เมื่อ NDBS_BUDGET_CONTROL ถูกเรียก จะมี NDBS_UpdateAllBudgetAmount ตามมาทันที
  • ทั้งหมดนี้เกิดขึ้นทั้งที่ผู้ใช้ยังไม่ได้เริ่มสร้างเอกสารหรือทำ transaction ใด ๆ
  • แต่ export ล่าสุดแสดงชัดแล้วว่า helper ปลายทางไม่ใช่เวอร์ชัน cursor / for loop เดิมอีกต่อไป

สรุปของ export ล่าสุดคือ login ยัง trigger budget flow เหมือนเดิม แต่คอขวดเดิมที่อยู่ใน NDBS_UpdateAllBudgetAmount ถูกแก้แล้ว ดังนั้นประเด็นที่ยังเหลืออยู่ตอนนี้คือ routing และ logic defect ใน procedure ย่อย มากกว่าจะเป็น row-by-row recalculation แบบเดิม

ทางทีม IT ได้แก้ไขประสิทธิภาพของ NDBS_UpdateAllBudgetAmount.sql เพื่อให้ใช้งานได้ตามปกติ

export ล่าสุดยืนยันแล้วว่า procedure นี้ไม่ใช่เวอร์ชัน cursor / for loop เดิมอีกต่อไป แต่ถูก rewrite ให้เป็น set-based bulk update แล้ว จึงอธิบายได้ตรงกับอาการจริงว่า หลังแก้ไฟล์นี้ ประสิทธิภาพกลับมาใช้งานได้ตามปกติ

แบบเดิม ช้าเพราะทำงานแบบทีละแถว

ใช้ cursor สองชุดวิ่งผ่าน budget department และ budget project ทั้งหมด แล้วในแต่ละรอบยังไป SELECT SUM และ UPDATE ซ้ำอีก ทำให้เกิดงานจำนวนมากแบบ row-by-row

แบบใหม่ เร็วขึ้นเพราะรวมงานเป็น bulk update

rewrite ให้เหลือการ UPDATE แบบ set-based สองก้อนใหญ่แทนการวนทีละแถว ทำให้ HANA ประมวลผลเป็นชุดเดียวและลด overhead จาก loop ซ้ำ ๆ ลงอย่างมาก

ดูโค้ดเปรียบเทียบแบบเดิม / แบบใหม่
แบบเดิม cursor + for loop + SELECT SUM/UPDATE ทีละแถว 
Declare cursor loopbudgetdept for Select * From "@NDBS_BGC_BDPL";
Declare cursor loopbudgetproj  for Select * From "@NDBS_BGC_BPJL";

for currloop as loopbudgetdept do
  Select SUM(IFNULL("Amount",0)) INTO NetReserve
  From "NDBS_BGC_OBDE"
  WHERE "Department" = currloop."U_Department"
    AND "BudgetGroup" = currloop."U_GroupCode"
    AND "BudgetYear" = currloop."Code";

  Update "@NDBS_BGC_BDPL" Set ...
end for;

for currloop as loopbudgetproj do
  Select SUM(IFNULL("Amount",0)) INTO NetReserve
  From "NDBS_BGC_OBPE"
  WHERE "Project" = currloop."U_Project"
    AND "BudgetGroup" = currloop."U_GroupCode"
    AND "BudgetYear" = currloop."U_Project";

  Update "@NDBS_BGC_BPJL" Set ...
end for;

  T."U_BudgetRes" = IFNULL((

    WHERE D."Department" = T."U_Department"

      AND D."BudgetYear"  = T."Code"
  ), 0),
  T."U_BudgetAct" = IFNULL((SELECT SUM(...) FROM "NDBS_BGC_OBDE" D ...), 0),
  T."U_BudgetBal" = IFNULL((SELECT SUM(...) FROM "NDBS_BGC_OBDE" D ...), 0),
  T."U_BudgetRem" = IFNULL(T."U_BudgetAmt", 0) - IFNULL((SELECT SUM(...) FROM "NDBS_BGC_OBDE" D ...), 0);

UPDATE "@NDBS_BGC_BPJL" T SET
